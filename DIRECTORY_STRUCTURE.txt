🎯 Layer Breakdown
===================

CORE LAYER (Centralized):
  • 2 Services (Auth, Project)
  • 2 Interfaces (User, Project)
  • 1 Config (API)
  • Services are singletons, injected throughout

SHARED LAYER (Reusable):
  • 3 Presentational Components
  • No service dependencies
  • Input/Output driven
  • Used in multiple features

FEATURES LAYER (Feature-specific):
  • 2 Page Components (Login, Dashboard)
  • Connected to services
  • Handle business logic
  • Route-specific

✅ Config:         [feature].config.ts
✅ Tests:          [name].spec.ts

🏗️ Architecture Pattern
========================

Smart Components (Page-level)
├─ Login Component
└─ Dashboard Component
   ├─ Sidebar Component (Dumb)
   ├─ Stats Component (Dumb)
   └─ Project-List Component (Dumb)

Services (Singleton)
├─ Auth Service
│  └─ Manages: Token, User, Headers
└─ Project Service
   └─ Manages: Projects, Loading State

State (Angular Signals)
├─ Auth State: token, currentUser
└─ Project State: projects, isLoading

🔄 Dependency Injection Pattern
=================================

Components use inject() to get services:
  private http = inject(HttpClient);
  private auth = inject(AuthService);

Services are provided at root level:
  @Injectable({ providedIn: 'root' })

Clean imports with barrel exports:
  import { AuthService } from '@app/core';
  import { SidebarComponent } from '@app/shared';
